# 作業計画とプロンプト履歴の管理

作業開始時に、tmp/ 配下に作業ごとのディレクトリを作成し、plan.md と prompt.md を作成して、作業中に適宜更新してください。/

## 使用方法

```txt
/start-implementation
{{作業内容を示すプロンプト、もしくは、プロンプトが書かれているファイルの参照}}
```

## ファイルの役割

- **plan.md**: 詳細な実行プラン（作業開始前に作成）
- **prompt.md**: ユーザーからの指示と、それに対する回答の履歴

## 作業の流れ

### 1. 作業開始時

1. tmp/{4 桁の連番}\_{作業名}/ ディレクトリの作成
2. plan.md の作成
3. prompt.md の作成

ディレクトリ名に使う 4 桁の連番は、現在存在するすべての作業ディレクトリ名を確認したうえで、
最大値に 1 を加えたものを使用してください。

例：サブテーブル対応の場合で、すでに tmp/0012_xxx のディレクトリが存在する場合

```
tmp/0013_subtable-support/
├── plan.md    # サブテーブル対応の実行プラン
└── prompt.md  # 作業履歴
```

- `prompt.md` はユーザーから指示があるたびに都度更新すること

### 2. プランニングの許可を得る

- コードの変更は勝手に実施してはいけません
- まず、`plan.md` に作業のプランニングを作成します
- `plan.md` をユーザーに提示し、許可を得ます
- 許可が得られるまで、ユーザーと相談しつつプランニングを繰り返します

### 3. 作業の実施

- `plan.md` に記載されたプランに従って、作業を実施します
- ユーザーから指示があるたびに `prompt.md` を更新します
- 作業完了後にも `prompt.md` を更新します
